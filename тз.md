\## 1. Общая информация о проекте



\### 1.1 Концепция и назначение платформы



«ПромОриентир» — цифровая платформа-агрегатор для подбора, сравнения и бронирования промышленных экскурсий по предприятиям России. Продукт позиционируется как «умный компас» в мире индустрии: он превращает восприятие завода из «скучного» в «технологичное и захватывающее».



Платформа объединяет три целевые аудитории:



\- \*\*Туристы / школьники / студенты / педагоги\*\* — конечные пользователи, ищущие экскурсии.

\- \*\*Предприятия (HR/PR/экскурсионные службы)\*\* — размещают паспорта объектов, управляют потоком заявок.

\- \*\*Администратор платформы (ТИЦ региона / команда проекта)\*\* — модерация, аналитика, управление данными.



\### 1.2 Якорный партнёр



Якорным партнёром и первым «реальным» предприятием на платформе является \*\*КССК (Кировский Сельский Строительный Комбинат)\*\*. Предприятие уже проводит промышленные экскурсии (подтверждено фотоматериалами). Данные КССК используются в seed-наполнении, включая интеграцию фото и видео через ВКонтакте (без загрузки медиа на сервер).



\### 1.3 Цели MVP



1\. Создать работающий прототип платформы с минимально необходимым набором функций.

2\. Протестировать на реальных пользователях (школьники, педагоги, сотрудники предприятий).

3\. Загрузить не менее 1 реального предприятия (КССК) + 5–7 сгенерированных для демонстрации.

4\. Обеспечить приём заявок на экскурсии и отправку уведомлений через ВКонтакте.

5\. Подключить LLM-ассистента через внешний API (OpenAI-совместимый).

\## 3. Функциональные требования



\### Модуль А: Интерактивная карта и каталог



\#### А.1 Карта России (главная страница)



\- На первом экране отображается интерактивная карта России (Яндекс Карты API).

\- Пользователь выбирает регион: кликом по карте или из списка под картой.

\- При выборе региона воспроизводится короткое видео-превью региона (15–20 сек, формат «рилс»). Видео встраивается через ВКонтакте iframe — загрузка на сервер не требуется.

\- После просмотра видео происходит переход на страницу региона: карта региона с маркерами предприятий + виджеты фильтрации слева.



\#### А.2 Каскадные фильтры



Каталог поддерживает 6 параметров фильтрации с каскадной логикой (AJAX, без перезагрузки страницы):



| \*\*Параметр\*\* | \*\*Тип\*\* | \*\*Значения\*\* |

|--------------|---------|---------------|

| Длительность | Выбор | 1 ч / 2 ч / полдня / полный день |

| Стоимость | Выбор | Бесплатно / до 500 руб. / 500–1500 руб. / 1500+ руб. |

| Размер группы | Выбор | 1–5 / 6–15 / 16–30 / 30+ |

| Тип производства | Мультивыбор | Машиностроение / Пищевое / Химическое / Энергетика / Строительное / IT-производство / Лёгкая промышленность / Другое |

| Образовательная программа | Мультивыбор | Профориентация / Углублённые знания / Знакомство с производством |

| Доступность ОВЗ | Мультивыбор | Нарушения зрения / Нарушения слуха / Нарушения ОДА |



При изменении любого фильтра система динамически обновляет доступные значения остальных фильтров (каскад), а также список карточек предприятий и маркеров на карте.



\#### А.3 Карточки предприятий в каталоге



\- Отображаются в виде сетки (3 колонки на десктопе, 2 на планшете, 1 на мобильном).

\- Каждая карточка содержит: превью-фото, название предприятия, тип производства, тэги, рейтинг интерактивности, краткое описание, кнопку «Подробнее».

\- Сортировка по умолчанию: по дате добавления (новые сначала). В MVP дополнительная сортировка не требуется.



\### Модуль Б: Карточка предприятия



\#### Б.1 Основная информация



\- Название, логотип, тип производства, регион, адрес.

\- Описание предприятия (текст, форматированный).

\- Фотогалерея (интеграция из ВКонтакте через iframe/oEmbed — без загрузки на сервер).

\- Видеоролик о предприятии (интеграция из ВКонтакте через iframe).

\- Ссылка на сайт / группу ВК предприятия.



\#### Б.2 Вкладка «Миф и реальность»



Специальная вкладка, доступная из карточки предприятия:



\- \*\*360°-тур:\*\* заглушка-плеер с текстом «Контент в разработке» + иконка. В будущем — iframe с WebXR/Marzipano-плеером.

\- \*\*AR-контент:\*\* аналогичная заглушка.

\- \*\*Виджеты сертификации:\*\* ISO, охрана труда, экостандарты — отображаются как иконки/бейджи, данные вводятся вручную предприятием.

\- \*\*«Живая статистика»:\*\* уровень шума, выбросов, аварий за 5 лет — таблица с данными, вводимыми вручную предприятием (числовые поля).

\- \*\*Сувенирная продукция:\*\* список с фото, описанием и ссылкой на приобретение (маркетплейс или сайт). Данные вводятся вручную.



\#### Б.3 Профориентационный блок



\- Список востребованных профессий предприятия (вводится вручную HR).

\- Навыки и возможности трудоустройства.

\- Данные о наставничестве, стажировках, социальном пакете.

\- Список текущих вакансий (обновляются при загрузке страницы из БД; HR обновляет вручную через ЛК).

\#### В.2 Подача заявки



Кнопка «Подать заявку» открывает форму (модальное окно или отдельная страница):



\- ФИО (обязательно)

\- Email (обязательно, валидация @)

\- Телефон (11 цифр)

\- Количество человек (число)

\- Желаемая дата (date picker)

\- Особые потребности (текстовое поле + чекбоксы ОВЗ)

\- Чекбокс «Ознакомлен с инструкцией по ТБ» (обязательный)



После подачи заявки:



1\. Заявка сохраняется в БД со статусом «новая».

2\. Отправляется уведомление предприятию и администратору через ВКонтакте (VK Notify API с токеном сообщества).

3\. Пользователю показывается экран «Заявка принята» + блок «Промышленная миля».



\#### В.3 Блок «Промышленная миля» (постзаявочный)



После подачи заявки пользователю показываются рекомендованные места поблизости (данные из БД):



\- Места размещения (гостиницы)

\- Предприятия питания (рестораны)

\- Развлечения (музеи, театры, парки, ТРЦ)



Отображается как карточки с названием, адресом, ссылкой на сайт/ВКонтакте.



\### Модуль Г: AI-ассистент «Smart Compass»



\#### Г.1 Общее описание



Интерактивный чат-помощник для подбора экскурсий в формате пошагового диалога (Wizard). Доступен с главной страницы и из каталога.



\#### Г.2 Промпт-стратегия ассистента



Системный промпт формируется динамически из следующих компонентов:



| \*\*Компонент системного промпта\*\* | \*\*Источник данных\*\* | \*\*Формат\*\* |

|----------------------------------|---------------------|-------------|

| Роль и контекст | Статичный текст в БД (редактируется в админке) | Текст |

| Каталог экскурсий (краткий) | Выборка из БД: id, название, тип, теги, стоимость, длительность | JSON-массив |

| Предпочтения пользователя (из диалога) | Накапливаются в ходе сессии | JSON-объект |

| Инструкции по формату ответа | Статичный текст в БД (редактируется в админке) | Текст |



\#### Г.3 Логика диалога



Пошаговый сценарий (Wizard):



1\. Приветствие + вопрос «Кто вы?» — Семья / Студент / Профессионал / Группа школьников.

2\. Вопрос «Что хотите увидеть?» — роботы / огонь / конвейер / чистые комнаты / дегустация / химия.

3\. Вопрос «Нужна ли еда/отель?» (да/нет).

4\. Система генерирует персональный шорт-лист (3–5 экскурсий) с пояснением, почему выбраны именно они.

5\. Пользователь может уточнить запрос в свободной форме — ответ обрабатывается LLM с контекстом предыдущих шагов.



\#### Г.4 Форматы данных для LLM



Каждая экскурсия передаётся в промпт в виде сжатого JSON-объекта:



```json

{"id": 1, "title": "Экскурсия на КССК", "type": "Строительное", "tags": \["бетон","строительство"], "cost": 0, "duration": "2ч", "interactivity": 8, "physical\_load": 5, "food": false, "capacity": 20}

```



Ответ ассистента содержит: массив id рекомендованных экскурсий + текстовое объяснение. Фронтенд подтягивает детали по id из локального состояния.



\#### Г.5 Fallback и ограничения



\- При недоступности LLM API — показывается сообщение «Ассистент временно недоступен, воспользуйтесь фильтрами».

\- Rate limit: не более 10 запросов к LLM в минуту от одного пользователя (ограничение на бэкенде).

\- История диалога хранится в sessionStorage фронтенда (не в БД в MVP), передаётся в каждый запрос как `messages\[]`.



\### Модуль Д: Личный кабинет предприятия



\- Регистрация и вход (JWT).

\- Заполнение/редактирование паспорта предприятия (все поля из Б.1–Б.3).

\- Управление экскурсиями: создание, редактирование, публикация, архивирование.

\- Входящие заявки: список с фильтром по статусу, возможность сменить статус (на рассмотрении / подтверждена / отклонена).

\- Аналитика: количество просмотров карточки, количество заявок по каждой экскурсии, динамика по дням (график).



\### Модуль Е: Панель администратора



\#### Е.1 Управление контентом



\- CRUD предприятий: создание, просмотр, редактирование, удаление, смена статуса (на модерации / опубликовано / заблокировано).

\- CRUD экскурсий: аналогичный функционал с возможностью принудительной публикации/снятия.

\- CRUD мест размещения и питания.

\- CRUD пользователей: просмотр списка, смена роли, блокировка.

\- Управление заявками: полный список со статусами, возможность смены статуса, экспорт в CSV.



\#### Е.2 Настройки LLM



В административной панели — раздел настроек AI-ассистента:



| \*\*Параметр\*\* | \*\*Тип\*\* | \*\*Описание\*\* |

|--------------|---------|---------------|

| API endpoint (base URL) | URL | OpenAI-совместимый эндпоинт |

| API ключ | Password-поле | Токен доступа к LLM API; хранится зашифрованным в БД |

| Модель (model name) | Текст | Например: gpt-4o, gpt-3.5-turbo, mistral-7b и т.д. |

| Системный промпт (часть 1: роль) | Textarea | Редактируемый текст роли ассистента |

| Системный промпт (часть 2: инструкции) | Textarea | Инструкции по формату ответа |

| Temperature | Float 0–2 | Степень случайности (по умолчанию 0.7) |

| Max tokens | Integer | Максимальный объём ответа (по умолчанию 1000) |

| Rate limit (req/min) | Integer | Лимит запросов от одного пользователя в минуту |



\#### Е.3 Настройки уведомлений ВКонтакте



| \*\*Параметр\*\* | \*\*Тип\*\* | \*\*Описание\*\* |

|--------------|---------|---------------|

| VK Community Token | Password-поле | Токен сообщества ВКонтакте для отправки уведомлений |

| VK Peer ID (получатель) | Число | ID чата/пользователя для уведомлений администратора |

| Шаблон уведомления о новой заявке | Textarea | Текст с переменными: {user\_name}, {tour\_name}, {date}, {count} |

| Шаблон уведомления о смене статуса | Textarea | Уведомление пользователю (если у него есть VK ID) |

| Уведомления включены | Toggle | Глобальный включатель уведомлений |



\#### Е.4 Аналитика (глобальная)



\- Общее количество предприятий / экскурсий / заявок.

\- Количество заявок по статусам.

\- Топ-10 экскурсий по просмотрам и заявкам.

\- Использование фильтров: какие параметры выбираются чаще (счётчики).

\- Графики динамики заявок и просмотров по дням (за последние 30 дней).



\### Модуль Ж: Сравнение экскурсий



\- Пользователь может добавить до 3 экскурсий в «сравнение» (иконка на карточке).

\- Страница сравнения: таблица с параметрами — уровень интерактивности (0–100%), физическая нагрузка, наличие питания, наличие сувениров, стоимость, длительность, доступность ОВЗ.

\- Визуализация: прогресс-бары или радарная диаграмма для интерактивности и физической нагрузки.

\- «Бонусная шкала»: наличие мерча, дегустации, уникальных фото-локаций — иконки/бейджи.

\## 8. Интеграции



\### 8.1 Яндекс Карты



\- Используется JavaScript API Яндекс.Карт для отображения интерактивной карты РФ.

\- Маркеры предприятий подгружаются через `/api/enterprises` (координаты: lat/lon в модели данных).

\- При клике на маркер — переход на карточку предприятия.

\- API-ключ Яндекс.Карт хранится в .env на сервере, передаётся фронтенду через эндпоинт `/api/settings/maps-key`.



\### 8.2 ВКонтакте — медиа-интеграция



\- Фото и видео предприятий встраиваются через iframe ВКонтакте.

\- Формат для фото: `https://vk.com/widget\_photos.php?oid={owner\_id}\&album\_id={album\_id}`

\- Формат для видео: `https://vkvideo.ru/video\_ext.php?oid={oid}\&id={id}\&hash={hash}`

\- В поля `vk\_photos\_url` и `vk\_video\_url` предприятие вводит полный iframe-код или ссылку — система встраивает через iframe на клиенте.



\### 8.3 ВКонтакте — уведомления



\- Уведомления отправляются через VK Community Messages API: `POST https://api.vk.com/method/messages.send`

\- Параметры: `access\_token` (токен сообщества из настроек), `peer\_id` (получатель), `message` (из шаблона), `random\_id` (уникальный), `v=5.199`

\- Вызов происходит на бэкенде при создании заявки (POST `/api/bookings`) и при смене статуса.

\- Если VK API недоступен или токен не задан — операция логируется, заявка всё равно сохраняется.



\### 8.4 LLM API



\- Используется OpenAI SDK с configurable base URL и API key (OpenAI-compatible).

\- Параметры (endpoint, model, ключ, промпты, temperature, max\_tokens) хранятся в таблице `settings`, редактируются в админке.

\- Запрос к LLM формируется на бэкенде: системный промпт + история диалога + текущий запрос пользователя.

\- В системный промпт динамически подставляется выборка экскурсий из БД (не более 50 объектов, сжатый JSON).



\### 8.5 Имитация оплаты (MVP)



\- Реальный платёжный шлюз не интегрируется.

\- В форме заявки — кнопка «Оплатить (тест)», при нажатии через 1–2 сек. `status` меняется на `paid` с сообщением «Оплата прошла успешно (тестовый режим)».

\- Поле `payment\_status` в таблице `bookings`: `pending` / `paid` / `refunded`.

\- Администратор может вручную сменить статус оплаты.

\### 9.1 КССК (якорное предприятие)



Первое и главное предприятие в seed-данных — КССК (Кировский Сельский Строительный Комбинат):



\- \*\*Название:\*\* КССК — Кировский Сельский Строительный Комбинат

\- \*\*Тип производства:\*\* Строительное

\## 12. Требования к производительности и безопасности



\### 12.1 Производительность



\- Время отклика API (95-й перцентиль): < 500 мс для стандартных запросов (без LLM).

\- Каскадная фильтрация: обновление результатов ≤ 300 мс (за счёт индексов на полях фильтрации).

\- LLM-ответ: не нормируется (зависит от внешнего API), но таймаут на бэкенде — 30 сек.

\- Поддержка одновременной работы 50+ пользователей в MVP (Docker + Express).



\### 12.2 Безопасность



\- JWT-токены: срок жизни access token — 24 ч. В MVP без refresh token.

\- Пароли: хеширование bcrypt (saltRounds = 12).

\- Чувствительные данные в settings (API-ключи, VK токен): хранятся в зашифрованном виде (AES-256 на ключе из .env).

\- CORS: разрешён только домен платформы (production) / localhost (development).

\- Rate limiting: не более 100 запросов/мин с одного IP (express-rate-limit), 10 LLM-запросов/мин на пользователя.

\- Валидация входных данных: все поля форм проверяются на сервере (express-validator или аналог).

\- SQL-инъекции: использование параметризованных запросов (prepared statements).

\## 14. Docker и деплой



Конфигурация по умолчанию:



\- Multi-stage Dockerfile (Alpine base image).

\- `docker-compose.yml`: сервис `app` + volume для SQLite (named volume `app\_data`) + bind mount для `./storage`.

\- `entrypoint.sh`: при старте автоматически инициализирует БД (`init-db.js`) и загружает seed-данные если БД пустая.

\- Порт публикации настраивается в `docker-compose.yml` (по умолчанию 8080 → 3000).



\*\*Команды запуска:\*\*



```bash

docker compose build   # Сборка образа

docker compose up -d   # Запуск в фоне

docker compose logs -f # Логи

docker compose down    # Остановка

```

\## 16. Критерии приёмки MVP



1\. Загружено не менее 1 реального предприятия (КССК) с фото/видео из ВКонтакте + 5 демо-предприятий.

2\. Работают каскадные фильтры по всем 6 параметрам (AJAX, без перезагрузки).

3\. Пользователь может подать заявку — уведомление приходит в VK (при наличии токена).

4\. AI-ассистент отвечает при наличии настроенного LLM API (graceful fallback при ошибке).

5\. Администратор может менять статусы заявок и редактировать контент через UI.

6\. Предприятие может зарегистрироваться и добавить экскурсию через ЛК.

7\. Платформа корректно отображается на мобильных устройствах (320px и выше).

8\. Вкладка «Миф и реальность» отображается с заглушками 360/AR и заполненными данными статистики.

9\. Страница сравнения экскурсий работает для 2–3 объектов.

10\. Блок «Промышленная миля» отображается после подачи заявки.

\## 17. Чего НЕ делать в MVP



\- Не разрабатывать мобильное приложение — только адаптивная вёрстка.

\- Не интегрировать реальный платёжный шлюз — только имитация.

\- Не интегрировать 1С, SAP, внешние HR-системы.

\- Не разрабатывать аудиогид и мобильный гид.

\- Не реализовывать геолокацию и построение маршрута до завода.

\- Не хранить медиа предприятий на своём сервере — только ВКонтакте iframe.

\- Не использовать ORM и better-sqlite3.

\- Не разрабатывать Refresh Token (JWT — только access token 24ч).

\- Не реализовывать автоматическую генерацию маршрута (экскурсии добавляются вручную).
